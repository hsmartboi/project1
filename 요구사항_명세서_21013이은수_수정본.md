### 개념 정의

뉴로모픽 컴퓨팅(Neuromorphic Computing)은 인간 뇌의 신경망 구조와 신호 원리를 모방하여 설계된 차세대 컴퓨팅 패러다임이다. 1980년대 Carver Mead가 최초로 실리콘 뉴런 및 시냅스 개념을 개척한 이후, 인공지능 연산 효율 향상의 핵심 기술로 주목받고 있다.

현대 컴퓨터는 연산 장치(CPU)와 메모리(RAM)가 물리적으로 분리된 **폰 노이만(von Neumann) 아키텍처**를 기반으로 하기 때문에, 데이터를 주고받는 과정에서 병목 현상이 발생하고 전력 소모가 크다는 단점이 있다. 뉴로모픽 컴퓨팅은 연산과 저장 기능을 동시에 수행하고, 필요한 시간에만 신호를 전달하는 **스파이킹(Spiking) 방식**으로 동작하여 에너지 효율을 극대화한다.

### 핵심 개념: 스파이킹 신경망 (SNN, Spiking Neural Network)

스파이킹 신경망은 뉴로모픽 컴퓨팅을 구현하는 핵심 알고리즘으로, **제3세대 인공신경망**으로 불린다. 기존 딥러닝(ANN, DNN)과 달리, 인간 뇌 신경 뉴런이 실제로 신호를 주고받는 방식인 **스파이크(Spike, 활동 전위)**를 직접 모사한다.

| 구분 | 기존 인공신경망 (ANN) | 스파이킹 신경망 (SNN) |
|-|-|-|
| 신호 형태 | 연속적인 실수값 | 이산적인 스파이크(0/1) |
| 정보 처리 방식 | 동기식 (Synchronous) | 비동기식, 이벤트 기반 |
| 에너지 효율 | 낮음 | 높음 (스파이크 발생 시에만 연산) |
| 시간 정보 활용 | 제한적 | 스파이크 타이밍으로 정보 인코딩 |

### 핵심 모델: LIF 뉴런 (Leaky Integrate-and-Fire)

LIF(Leaky Integrate-and-Fire) 모델은 뉴로모픽 시뮬레이션에서 가장 널리 사용되는 뉴런 모델이다. 방정식은 다음과 같다:

```
τ_m · (dV/dt) = -(V - V_rest) + R · I(t)
```

* `V` : 막전위 (Membrane Potential)
* `V_rest` : 안정 전위 (Resting Potential)
* `τ_m` : 시상수 (Time Constant)
* `R` : 막 저항, `I(t)` : 입력 전류
* 막전위가 역치(`V_threshold`)를 초과하면 스파이크를 발생시키고, 전위를 리셋

### 학습 규칙: STDP (Spike-Timing-Dependent Plasticity)

STDP는 뉴로모픽 시스템의 핵심 학습 알고리즘으로, 스파이크의 **타이밍 차이**에 따라 시냅스 가중치를 강화하거나 약화시킨다. 입력 스파이크가 출력으로 뉴런 간 연결이 강화되는 뇌의 학습 원리를 직접 모사한다.

정량적 수식은 다음과 같다:

```
Δw = A₊ · exp(-Δt / τ₊)    if Δt > 0  (사전 뉴런이 먼저 발화 → 가중치 강화)
Δw = -A₋ · exp( Δt / τ₋)   if Δt < 0  (사후 뉴런이 먼저 발화 → 가중치 약화)

여기서 Δt = t_post - t_pre
```

| 파라미터 | 기본값 | 의미 |
|-|-|-|
| `A₊` | 0.01 | 강화 학습률 |
| `A₋` | 0.01 | 약화 학습률 |
| `τ₊` | 20 ms | 강화 시상수 |
| `τ₋` | 20 ms | 약화 시상수 |

## 용어 정의

| 용어 | 정의 |
|-|-|
| **Spike (스파이크)** | 뉴런이 역치를 초과할 때 발생하는 순간적인 전기 신호 (활동 전위) |
| **LIF** | Leaky Integrate-and-Fire. 뉴런이 전류를 누적하다가 역치 도달 시 발화하는 단순화 모델 |
| **SNN** | Spiking Neural Network. 스파이크를 신호로 사용하는 신경망 |
| **STDP** | Spike-Timing-Dependent Plasticity. 스파이크 타이밍 차이에 따른 시냅스 가중치 업데이트 규칙 |
| **막전위 (V)** | 뉴런 세포막 내외의 전위차 (단위: mV) |
| **역치 (V_threshold)** | 스파이크가 발생하는 막전위의 임계값 |
| **불응기 (Refractory Period)** | 스파이크 발생 직후 추가 스파이크가 억제되는 기간 |
| **시냅스 가중치 (w)** | 뉴런 간 연결 강도를 나타내는 수치 |

## 4장. 테스트 계획 (Test Plan)

### 4.1 기능 요구사항 → 단위 테스트 (Unit Test)

각 기능 모듈을 독립적으로 검증한다. 아래 테스트 케이스를 모두 통과해야 기능 구현이 완료된 것으로 판단한다.

#### [UT-01] LIF 뉴런 막전위 계산 검증 (F-01 대상)

| 항목 | 내용 |
|-|-|
| **테스트 목적** | 입력 전류가 0일 때 막전위가 안정 전위로 수렴하는지 확인 |
| **입력 조건** | `I = 0 nA`, `V_init = -65 mV`, `V_rest = -70 mV`, `t = 100 ms` |
| **기대 출력** | `V(t)` 값이 시간에 따라 `V_rest = -70 mV`에 지수적으로 수렴 |
| **합격 기준** | `t = 100 ms` 시점에서 `\|V(t) - V_rest\| < 0.1 mV` |

> ✅ **수정 사항**: 원본에서 합격 기준이 누락되어 있었음 → 복원 완료

---

#### [UT-02] 스파이크 발생 및 리셋 검증 (F-02 대상)

| 항목 | 내용 |
|-|-|
| **테스트 목적** | 충분한 입력 전류 공급 시 스파이크 발생 및 리셋이 정상 동작하는지 확인 |
| **입력 조건** | `I = 2.0 nA` (역치 초과 유발), `V_threshold = -55 mV`, `V_reset = -75 mV` |
| **기대 출력** | 시뮬레이션 동안 스파이크 최소 1회 이상 발생, 스파이크 직후 `V = V_reset` |
| **합격 기준** | `len(spike_times) >= 1` 이고 리셋 직후 `V == V_reset` |

---

#### [UT-03] 불응기 동작 검증 (F-03 대상)

| 항목 | 내용 |
|-|-|
| **테스트 목적** | 스파이크 발생 직후 불응기 동안 추가 스파이크가 억제되는지 확인 |
| **입력 조건** | `I = 10.0 nA` (강한 전류), `refractory_period = 2 ms` |
| **기대 출력** | 연속된 두 스파이크 간격이 불응기(2 ms) 이상 |
| **합격 기준** | `min(diff(spike_times)) >= refractory_period` |

---

#### [UT-04] STDP 가중치 업데이트 방향 검증 (F-05 대상)

| 항목 | 내용 |
|-|-|
| **테스트 목적** | 사전 뉴런이 먼저 발화하면 가중치가 강화되는지, 나중에 발화하면 약화되는지 확인 |
| **입력 조건 (강화)** | `t_pre = 10 ms`, `t_post = 12 ms` → `Δt = +2 ms` |
| **입력 조건 (약화)** | `t_pre = 12 ms`, `t_post = 10 ms` → `Δt = -2 ms` |
| **기대 출력** | `Δt > 0` → `Δw > 0` (가중치 증가), `Δt < 0` → `Δw < 0` (가중치 감소) |
| **합격 기준** | 두 경우 모두 부호가 기댓값과 일치 |

---

#### [UT-05] 스파이크 래스터 플롯 출력 검증 (F-07 대상)

| 항목 | 내용 |
|-|-|
| **테스트 목적** | 시뮬레이션 결과 래스터 플롯이 오류 없이 출력되는지 확인 |
| **입력 조건** | 뉴런 10개, `t = 200 ms`, 각 뉴런 입력 전류 랜덤 설정 |
| **기대 출력** | matplotlib 그래프가 오류 없이 출력되고, 각 뉴런의 스파이크가 정확한 시점에 표시됨 |
| **합격 기준** | 코드 실행 시 예외(Exception) 없이 그래프 파일 저장 성공 |

---

#### [UT-06] 유효성 검사 동작 검증 (F-10 대상)

| 항목 | 내용 |
|-|-|
| **테스트 목적** | 잘못된 파라미터 입력 시 `ValueError`가 발생하는지 확인 |
| **입력 조건** | `V_threshold = -80 mV` (`V_rest = -70 mV`보다 낮은 값), `dt = -0.1` |
| **기대 출력** | `ValueError` 예외 발생 및 오류 메시지 출력 |
| **합격 기준** | `pytest.raises(ValueError)` 또는 `try-except`로 예외 확인 |

### 4.2 비기능 요구사항 → 통합 테스트 (Integration Test)

기능 모듈을 통합하여 전체 시스템이 비기능 요구사항을 만족하는지 검증한다.

#### [IT-01] 성능 통합 테스트 (NF-01 대상)

| 항목 | 내용 |
|-|-|
| **테스트 목적** | 전체 SNN 시뮬레이션의 실행 시간 검증 |
| **테스트 방법** | `time.perf_counter()`로 전체 실행 시간 측정. 뉴런 100개, 시뮬레이션 1000 ms 기준 |
| **기대 결과** | 총 실행 시간 ≤ 30초 |
| **합격 기준** | 3회 반복 실행 평균 ≤ 30초 |

---

#### [IT-02] 재현성 통합 테스트 (NF-04 대상)

| 항목 | 내용 |
|-|-|
| **테스트 목적** | 동일 시드로 2회 실행 시 동일한 결과가 나오는지 확인 |
| **테스트 방법** | `numpy.random.seed(42)` 고정 후 2회 실행, 스파이크 발생 횟수 및 최종 가중치 행렬 비교 |
| **기대 결과** | 두 실행 결과의 스파이크 타이밍 배열이 완전히 일치 |
| **합격 기준** | `numpy.array_equal(result_1, result_2) == True` |

---

#### [IT-03] 엔드-투-엔드 시각화 통합 테스트 (NF-01, NF-06 대상)

| 항목 | 내용 |
|-|-|
| **테스트 목적** | LIF 구현 → SNN 시뮬레이션 → STDP 학습 → 시각화까지 전체 파이프라인 검증 |
| **테스트 방법** | 메인 스크립트를 처음부터 끝까지 실행하여 그래프 파일 3종 (막전위, 래스터 플롯, 가중치 변화) 생성 확인 |
| **기대 결과** | 3종의 `.png` 파일이 지정 폴더에 생성됨, 각 파일 해상도 ≥ 150 DPI |
| **합격 기준** | 파일 존재 확인 및 `matplotlib` figure 객체의 `fig.get_dpi() >= 150` 으로 DPI 검증 |

> ✅ **수정 사항**: 원본에서 `PIL` 라이브러리로 DPI 검증 → 제약 조건(NF-05)의 허용 라이브러리(numpy, matplotlib)만 사용하도록 `fig.get_dpi()` 방식으로 변경

---

#### [IT-04] 모듈 통합 및 인터페이스 테스트 (NF-03 대상)

| 항목 | 내용 |
|-|-|
| **테스트 목적** | LIF 뉴런 클래스, SNN 네트워크 클래스, 시각화 모듈 간 데이터 인터페이스 적합성 검증 |
| **테스트 방법** | 각 모듈의 출력 데이터 타입 및 형태(shape)를 assert로 검증. 예: `spike_times`가 list 타입인지, `weight_matrix`가 (N×M) numpy array인지 확인 |
| **기대 결과** | 모든 assert 통과, 모듈 간 데이터 전달 시 타입 오류 없음 |
| **합격 기준** | `AssertionError` 발생 없이 전체 통합 실행 완료 |
